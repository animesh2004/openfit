Why runllm

Honest answers, not guesses.

Most “can I run it?” tools give one made-up number. runllm models how you actually run LLMs.

Quant-aware sizing

Real RAM at Q4_K_M, Q8_0 and FP16 — the way Ollama runs models — not a single full-precision figure.

GPU / VRAM verdict

Knows the difference between ⚡ fast (fits in VRAM) and 🐢 slow (RAM-only) — the real usability line.

Verified ollama commands

Only shows ollama pull commands that actually exist in Ollama's library. No fake tags that 404 on first try.

MCP-native

Call it from Claude Code, Cursor, VS Code, Windsurf or Zed — three tools, zero glue code.

Filter by use case

Text, Code, Vision, Image-gen or Audio — filtered on real OpenRouter modality metadata, free or paid.

Scriptable

Flags + --json for CI and pipelines: runllm --free --code --json. No prompts, machine-readable.

How it works

From terminal to a ranked list

Pick tier & use case

Free/Paid and Text / Code / Vision / Image / Audio — by prompt or flags.

Scan hardware

RAM, CPU, GPU and VRAM detected via systeminformation.

Compute per-quant footprint

params × bytes-per-weight × overhead at Q4 / Q8 / FP16. MoE models sized by total params (all experts resident).

Match against RAM + VRAM

Fits in VRAM → fast. Fits in RAM only → slow. Doesn't fit even at Q4 → can't run.

Get your list

Runnable models with a verified ollama pull — plus what you'd need for the rest.
