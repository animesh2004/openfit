Why openfit

Stop guessing. Start running.

No more pulling a 40GB model only to watch it OOM. openfit checks your hardware first.

Hardware-aware

Reads total/free RAM via os, your CPU model, and GPU (system_profiler, nvidia-smi or wmic) — silently, behind a spinner.

Live OpenRouter data

Fetches every model from the public OpenRouter API — no key required — then filters by free/paid and your use case.

Copy-paste ready

Every model you can run comes with the exact ollama pull command. No mental math on quantization.

Instant & zero-setup

One npx command. No install, no config, no account. Runs in seconds and cleans up after itself.

How it works

Four steps, one command

From terminal to a ranked list of runnable models.

Pick a tier

Choose Free or Paid models with the arrow keys.

Pick a use case

Text / Chat, Code, Vision, Image generation, Audio, or Any.

openfit scans & matches

It detects your hardware, estimates each model's RAM need from its parameter size, and compares.

Get your list

Two lists — what you can run (with ollama pull) and what you can't (with how much more RAM you'd need).

Filter by use case
› Text / Chat
› Code
› Vision
› Image generation
› Audio
› Any

Try it right now

It takes one command and about five seconds.

$ npx @_abstel/openfit

No local RAM? Run any model via the OpenRouter API.
